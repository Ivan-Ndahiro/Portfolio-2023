UX Redesign: Better UK - An health and gym application

< UI/UX

Product Overview:

The Better UK app is a mobile application for users to book activities and buy memberships for Better UK gyms across the country. As a user of this app, I discovered a problem regarding the user interface when first understanding what features were avaliable and where I could find them.



Problem:

In the original design, all the options are homogenised in a grid format and feels too cramped. While the booking options are higher on the page and larger ever so slightly, they repeat the term “Book” and blend in too well with the other grid icons.


Solution:

I intended to improve the general user experience of the home page by reducing any clutter to simplify the user interface. By sectioning the major options for the user, it should improve the efficiency for access and bring a higher level of openness. I also wanted to bring a modern approach to the UI design.


Tools & Programs Used:


  • Figma Desktop for UI design
  • Additional Icon Set by Iconduck for additional symbols

    • Goals:


  • Create faster access to view bookings and account options
  • Reduce excess text on the home page
  • Keep initial visual identity

    • Wireframes:

    For my initial concept, I desigend a quick wireframe for where I wanted the options to be placed to improve the user's understanding of the app while keeping the original visual language. I wanted to segment the different features more clearly and chose to make a new section called "Your Gym" to align with the new layout.


    After developing a refined wireframe, I revised the many of the sections and design additions. Firstly, I removed the "Your Gym" heading as the apps bubble layout stood out better. To better distinguish each section, I made sure to keep them uniform-yet-distinct with different shapes and sizes.


    Final Design:

    With the final design, I decided to re-organise the order of the Booking and My Account sections as after revaluation, I understood users would come to the app requiring access to the former on a more regular basis than the latter. There is also more space on the redesign, giving the user a less cramped experience.


    Takeaways:

    For a homepage, good accessibility is important and creating a positive first impression is critical. As the application is meant for many repeat users, the ease-of-access to book activities is significant. Overall, I learned what good and bad features the initial design had and understood what to keep and remove to improve its usability.
